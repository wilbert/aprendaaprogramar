# Aprenda a Programar

An interactive Ruby programming tutorial — the PT-BR translation of Chris Pine's
*Learn to Program* — served as a **Rails 8** application running on **Ruby 3.4.4**
and **Puma**.

The tutorial pages are generated by a self-contained legacy engine under
[`lib/learn_to_program_tutorial`](lib/learn_to_program_tutorial). Every code sample
you see is actually executed by Ruby on each request, so its output is real. A thin
Rails controller adapts that engine (originally written against Ruby's CGI class)
to modern Rails.

## Stack

- Ruby 3.4.4
- Rails 8.1
- Puma 8
- No database and no asset pipeline — the app renders HTML directly and serves its
  static CSS/images from [`public/`](public).

## Project layout

- [`app/controllers/learn_controller.rb`](app/controllers/learn_controller.rb) —
  serves the tutorial. It wraps request params in a small CGI-compatible shim so the
  legacy engine runs unchanged, and serializes rendering with a mutex (the engine
  swaps `Kernel#puts`/`#gets` while running sample code and isn't reentrant).
- [`lib/learn_to_program_tutorial/`](lib/learn_to_program_tutorial) — the tutorial
  content/engine (loaded explicitly in
  [`config/initializers/learn_to_program.rb`](config/initializers/learn_to_program.rb),
  not via Zeitwerk, because it predates modern autoloading conventions).
- [`config/`](config) — a minimal Rails 8 config that loads only Action Controller
  and Action View (no Active Record / Active Job / Action Mailer).
- [`deploy/`](deploy) — systemd unit, nginx config, and sudoers snippet for the VPS.

## Local development

```bash
# Ruby 3.4.4 is pinned in .ruby-version (use rbenv/rvm/asdf).
bundle install
bin/rails server        # http://localhost:4050
```

Useful URLs:

- `/` — the tutorial home / table of contents
- `/index.rb?Chapter=01` — a specific chapter (`00`–`11`)
- `/up` — health check

### Tests

```bash
bin/rails test
```

The integration test ([`test/integration/tutorial_test.rb`](test/integration/tutorial_test.rb))
renders every chapter and asserts the engine never raises.

## Deployment

Production runs on a Contabo VPS (the same host as the `mio` project), deployed with
**Capistrano** — Puma runs as a systemd service behind nginx, which proxies to a unix
socket. See [DEPLOYMENT.md](DEPLOYMENT.md) for one-time server setup and the deploy
workflow. In short:

```bash
bundle exec cap production deploy
```
